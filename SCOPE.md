# PSYCHE/OS Scope

This file is the reality check for the repository, not a pitch deck.

## Solid Today

### Interface
- A strong typographic dashboard for exploring a psyche analysis artifact
- Views for overview, patterns, dimensions, archetypes, potentials, narrative, semantic map, integration, and setup
- Local screenshot-ready UI suitable for a public repository

### Repo Baseline
- Publishable open-source hygiene: license, contributing, security policy, CI, issue templates, deployment notes
- Local-first defaults that keep sensitive source data and generated outputs out of git
- Frontend build, lint, and validation checks that pass

### Pipeline Surfaces
- Runnable shell scripts for extraction, synthesis, neurodivergence screening, and vector embeddings
- Canonical prompt files shared by UI and shell workflows
- Output conventions that are good enough to iterate in public

## Real But Still Evolving

- Prompt quality, calibration, and cross-source synthesis depth
- Source adapter compatibility and normalization
- Evaluation and regression testing for prompt outputs
- Directional-vector quality and longitudinal usefulness
- TypeScript core pipeline, which still contains scaffolded/stubbed logic in parts

## Not Done

- End-to-end validated extraction across a broad adapter matrix
- Hard guarantees that all generated JSON conforms to a stable schema
- Hosted deployment verification across multiple providers
- Production-ready analytics, ontology versioning, or graph persistence
- Formal claims of diagnostic, psychometric, or therapeutic validity

## Next Execution Focus

The highest-value work right now is:

1. strengthen prompt design and reduce fragmentation
2. expand source compatibility without lowering quality
3. add evaluation fixtures and quality gates
4. improve the directional-vector output so it points to coherent movement instead of cheap advice
5. keep the interface clean while the analysis engine catches up
